# Contributing

Thanks for your interest in improving Virtual Pro Wrestling 2: Recompiled!

## Ground rules

- **Never commit ROM data or extracted game assets.** PRs containing game data will be
  closed. CI builds fetch the ROM from a private repository; fork PRs build without it.
- This project is licensed under GPLv3 (see `COPYING`); contributions are accepted
  under the same license.
- Start with `BUILDING.md` (how to build). For background on the techniques and
  invariants this port is built on, the sister projects are the best reference:
  [WCW vs. nWo World Tour: Recompiled](https://github.com/jessetbh/WCWvsNWOWorldTourRecomp)
  (`docs/devlog.md` there),
  [WCW/nWo Revenge: Recompiled](https://github.com/jessetbh/WCWnWoRevengeRecomp),
  [WWF WrestleMania 2000: Recompiled](https://github.com/jessetbh/WWFWrestleMania2000Recomp),
  and
  [Virtual Pro Wrestling 64: Recompiled](https://github.com/jessetbh/VPW64Recomp) —
  the five ports share their runtime stack, and VPW2's libultra is being
  identified by fingerprint transfer from WrestleMania 2000's.

## Practical notes

- The runtime libraries (N64ModernRuntime, RecompFrontend, RT64) carry project-local
  fixes tagged `[wcw fix]` / `[wcw2k]` on the `wcw` branches of the jessetbh forks.
  If you change anything under `lib/`, the change belongs in the corresponding fork
  (PR against its `wcw` branch); the submodule pin here is then bumped to the pushed
  commit.
- **Never hand-edit recompiler output** (`RecompiledFuncs/` is generated and
  gitignored). Diagnostics that must live in generated files are applied by
  `tools/readd_hand_edits.py` — extend that script instead.
- **After any recompiler regeneration, the full post-regen pipeline is mandatory**
  (see `BUILDING.md`): `fix_stumps.py`, `fix_switches.py`, `fix_backbranches.py`
  (loop until "0 missing"), `readd_hand_edits.py`, `audit_coresidency.py`. Each one
  guards against a silent-miscompile class this game actually hit.
- Symbol identifications (naming a `func_XXXX`) need evidence — add it to
  `disasm/libultra.md` alongside the `RENAME` entry in `tools/gen_symbols.py`.
- Symbol metadata is tracked in-repo: `syms/dump.toml` is generated by
  `tools/gen_symbols.py` from the splat disassembly (`disasm/`) — regenerate it,
  don't hand-edit.
- Test on a real build before opening a PR: boot, play a match **through the
  post-match transition back to the menus**, and open the config menu.

## Reporting bugs

Use the issue template. Always include your GPU + driver version, OS, the log output,
and your ROM's SHA1 (the only supported ROM is the USA Rev 1 release, SHA1
`82DD25A044689EAB57AB362FE10C0DA6388C217A`).
